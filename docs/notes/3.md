
将回调风格的 `getImageAsDataUrl` 重构为 Promise 风格，不仅能消除“回调地狱”，还能更优雅地处理错误（如图片加载失败、Canvas 污染等）。

以下是优化后的代码。我将逻辑整合到了一个统一的 Promise 函数中，并增加了**错误处理**和**超时保护**。

### 优化后的代码

```javascript
/**
 * 将 <img> 转为 Data URL（Promise 风格）
 * 特点：
 * 1. 自动处理图片未加载完成的情况（监听 load 事件）
 * 2. 自动处理图片加载失败的情况（监听 error 事件）
 * 3. 包含超时保护，防止因图片永远不加载导致 Promise 挂起
 * 4. 返回完整的结果对象 { url, code, starName }
 */
function getImageAsDataUrlPromise(timeoutMs = 5000) {
  return new Promise((resolve, reject) => {
    const code = getCode();
    const star = getStar();
    const starName = star.length === 1 ? star[0]?.name : 'todo';
    
    const imgEl = document.querySelector('.screencap img');

    // 1. 基础检查：元素是否存在
    if (!imgEl) {
      return reject(new Error('Image element (.screencap img) not found'));
    }

    // 2. 定义核心处理逻辑
    const processImage = () => {
      try {
        // 再次检查完整性（以防竞态条件）
        if (!imgEl.complete || imgEl.naturalWidth === 0) {
          return reject(new Error('Image loaded but invalid or empty'));
        }

        const canvas = document.createElement('canvas');
        const ctx = canvas.getContext('2d');
        
        // 设置画布尺寸
        canvas.width = imgEl.naturalWidth || imgEl.width;
        canvas.height = imgEl.naturalHeight || imgEl.height;
        
        // 绘制图片
        ctx.drawImage(imgEl, 0, 0);

        // 转换为 Blob
        canvas.toBlob((blob) => {
          if (!blob) {
            return reject(new Error('Failed to create blob from canvas'));
          }

          const reader = new FileReader();
          
          reader.onload = () => {
            resolve({ 
              url: reader.result, 
              code, 
              starName 
            });
          };
          
          reader.onerror = () => {
            reject(new Error('FileReader failed to read blob'));
          };

          reader.readAsDataURL(blob);
        }, 'image/jpeg', 0.95);

      } catch (err) {
        reject(err);
      }
    };

    // 3. 状态判断与事件绑定
    if (imgEl.complete && imgEl.naturalWidth !== 0) {
      // 情况 A: 图片已加载完成，直接处理
      // 使用 setTimeout 确保异步执行，保持 Promise 行为一致
      setTimeout(processImage, 0);
    } else if (imgEl.error) {
      // 情况 B: 图片已经加载失败
      reject(new Error(`Image failed to load: ${imgEl.src}`));
    } else {
      // 情况 C: 图片正在加载中，绑定事件
      const onLoad = () => {
        cleanup();
        processImage();
      };

      const onError = () => {
        cleanup();
        reject(new Error(`Image failed to load during wait: ${imgEl.src}`));
      };

      imgEl.addEventListener('load', onLoad, { once: true });
      imgEl.addEventListener('error', onError, { once: true });

      // 清理函数：移除监听器，防止内存泄漏或重复触发
      const cleanup = () => {
        imgEl.removeEventListener('load', onLoad);
        imgEl.removeEventListener('error', onError);
        clearTimeout(timeoutTimer);
      };

      // 4. 超时保护
      const timeoutTimer = setTimeout(() => {
        cleanup();
        reject(new Error(`Timeout: Image did not load within ${timeoutMs}ms`));
      }, timeoutMs);
    }
  });
}
```

### 主要改进点说明

1.  **统一入口**：不再需要 `getImageAsDataUrl` (回调版) + `getImageAsDataUrlPromise` (包装版) 两个函数。直接使用 `getImageAsDataUrlPromise` 即可。
2.  **完善的错误处理**：
    *   增加了 `imgEl.error` 检查，如果图片在调用前就已经坏了，立即拒绝 Promise。
    *   监听了 `error` 事件，如果等待过程中图片加载失败，会正确抛出错误，而不是像旧代码那样永远等待 `load`。
    *   增加了 `try...catch` 包裹 Canvas 操作，防止因 CORS 问题（"Tainted canvas"）或其他运行时错误导致脚本崩溃。
3.  **超时机制 (Timeout)**：
    *   新增了 `timeoutMs` 参数（默认 5 秒）。如果图片因为网络问题或资源失效导致永远不触发 `load` 或 `error`，Promise 会在指定时间后自动 Reject，防止程序卡死。
4.  **资源清理**：
    *   使用了 `cleanup` 函数，在成功、失败或超时后，都会移除 `load` 和 `error` 监听器并清除定时器，避免内存泄漏。
5.  **异步一致性**：
    *   即使图片已经加载完成 (`imgEl.complete`)，也使用了 `setTimeout(..., 0)` 将其放入微任务队列，确保无论哪种情况，Promise 的 resolve 都是异步发生的，这符合 Promise 的标准行为规范，避免同步 resolve 导致的潜在调用栈问题。

### 关于 CORS 的特别提示
代码注释中提到“绕过 CORS”。请注意：
*   **如果图片来自同源**：此代码工作完美。
*   **如果图片来自跨域 (Cross-Origin)**：
    *   如果服务器设置了 `Access-Control-Allow-Origin: *` (或你的域名)，且你在 `<img>` 标签上添加了 `crossorigin="anonymous"` 属性，此代码可以正常工作。
    *   如果服务器**没有**设置 CORS 头，或者图片被标记为 "Tainted" (污染)，`ctx.drawImage` 虽然能执行，但随后的 `canvas.toBlob()` 或 `toDataURL()` 会抛出安全错误 (`SecurityError: The operation is insecure`)。
    *   **无法通过纯前端 JS 绕过浏览器的安全策略**。如果遇到这种情况，必须通过后端代理（Proxy）来请求图片，或者确保图片服务器配置了正确的 CORS 头。上面的 `try...catch` 块会捕获这个错误并reject Promise。
